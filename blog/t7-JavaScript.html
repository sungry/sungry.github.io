Ruby vs. Javascript

Ruby Hashes vs. JavaScript Objects - Ruby classes vs. JavaScript constructor functions - Ruby documentation vs. JavaScript documentation - Looping in Ruby vs. Looping in JavaScript I've learned about Ruby for about 4 weeks now and also just started to taste on Javascript. The both languages are very powerful and used widespreadly. Since I am learning the two languages simultaneously, I thought it'd be a good idea to compare those two to clear things out! The topic I've chosen to talk about here is loops. Even though both languages' loop syntaxes essentially do the same thing, they could look quite different.

JavaScript

Essentially, Javascript follows the traditional loop syntax. For for loops, this is what it looks like. 
        
      for (var i = 1; i <= 5; i++) {
        console.log("loop  " + i);
      }
        
      
      First, var i is a temporary variable one uses for the loop because every for loop needs a starting point. In this code, we are starting at a value of 1. i <= 5 sets a limit, otherwise the loop will go on infinitely. Here it says, if i is less than or equal to 5, stop. i++ means i = i + 1. meaning, each time loop goes through, i increments itself. For example, in the first loop i is 1, then next loop i becomes 2 and so forth.

      During each loop, it prints out currently value of i.

      
Ruby

      Although Ruby does work with the same way Javascipt does looping. It has its own neat and convenient loop syntax.

      Let's say there's an array and it has 5 integer elements and we want to print that out each time.

      
        
      array = [1,2,3,4,5]

      array.each { |current_value| puts current_value }
        
      


      I know, it look scompletely different! For Javascript we would've used array.length instead of i <= 5 for the limit.
      Anyways, what the each method does is that it literally iterates through 'each' element of the loop. Each time this it iterates an element in the array, it prints out its element.
